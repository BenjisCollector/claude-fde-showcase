# Forward Deployed Engineering

A Forward Deployed Engineer embeds with a customer to design, build, and ship
production systems that turn frontier model capabilities into concrete business
outcomes. The role sits at the boundary between engineering and the customer:
part software engineer, part solutions architect, part trusted advisor.

The work is end to end. You scope the problem with the customer, prototype fast
to prove value, then harden the prototype into something that runs reliably
against real data and real users. You write the integration code, the tools,
the evaluation harness, and the runbook. You also translate: turning a vague
business goal into a precise technical specification, and turning model
behaviour back into language a stakeholder can act on.

Three engineering primitives recur constantly in this work. MCP servers give
the model typed, auditable access to a customer's systems. Sub-agent
orchestration breaks a hard task into testable, swappable units. Agent skills
capture domain knowledge as reusable, self-describing capabilities. This
repository implements all three with real, runnable, offline logic so the
pattern is concrete rather than aspirational.
